# 参与贡献

感谢你参与 EaseGPT。提交贡献即表示你同意按照项目的 Apache License 2.0
提供该贡献，并确认你有权提交相关代码、文档和资源。

## 开发流程

1. 从最新主分支创建功能分支。
2. 保持改动聚焦，不提交数据库、密钥、日志、上传文件或构建产物。
3. 使用 UTF-8 编码，避免引入乱码。
4. 执行 `dotnet restore` 和 `dotnet build --configuration Release`。
5. 为行为变更补充测试或说明验证步骤。
6. 提交 Pull Request，并填写变更目的、验证结果和兼容性影响。

## 提交要求

- 不得提交 API Key、访问令牌、连接字符串或真实用户数据。
- 新增依赖和素材必须注明来源及许可证。
- 修改已有文件时保留原有版权和归属声明。
- 对外行为或配置发生变化时同步更新 README 或相关文档。
- 安全漏洞请按 `SECURITY.md` 私下报告，不要创建公开 Issue。

Apache License 2.0 第 5 节提供，不附加额外条款。
