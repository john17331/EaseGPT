# 安全策略

## 支持范围

安全修复优先应用于默认分支和最新正式版本。尚未正式发布的开发版本不承诺长期
维护，但确认的漏洞仍会尽快处理。

## 报告漏洞

请不要通过公开 Issue 报告漏洞或泄露可利用细节。GitHub 仓库启用后，优先使用
仓库的 **Security > Report a vulnerability** 私密报告功能；如未启用，请通过
维护者在仓库中公布的安全联系邮箱报告。

报告建议包含：

- 受影响的版本、提交或部署方式。
- 可复现步骤和最小复现材料。
- 影响范围及可能的利用方式。
- 建议修复方案（如有）。

请勿在未经授权的系统或真实用户数据上进行测试。维护者会确认收到报告，并在
完成风险评估后协调修复和披露时间。

## 凭据泄露

若发现仓库历史、日志或发行包中包含真实密钥，请立即吊销并轮换密钥。仅删除
当前文件不足以消除 Git 历史和已下载副本中的泄露风险。
